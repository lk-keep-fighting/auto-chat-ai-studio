# v1.5.0 快速参考

> 交互式错误处理和循环执行

---

## 🚨 遇到错误时

### 操作选项

| 输入 | 说明 | 使用场景 |
|------|------|----------|
| **1-25** | 从指定步骤继续 | 跳过或重新执行特定步骤 |
| **retry** | 重试当前步骤 | 临时错误，重试可能成功 |
| **skip** | 跳过当前视频 | 视频有问题，无法处理 |
| **quit** | 退出程序 | 需要修改代码或配置 |
| **Enter** | 继续下一步 | 已手动处理，继续执行 |

---

## 🔄 批次完成后

### 操作选项

| 输入 | 说明 |
|------|------|
| **continue** 或 **Enter** | 重新加载 VideoList.csv，继续下一批 |
| **quit** | 退出程序 |

---

## 📋 常见场景

### 网络超时
```
❌ 错误: 上传视频超时
👉 输入: retry
```

### AI 响应异常
```
❌ 错误: 步骤 15 失败
👉 输入: 15 (重新执行步骤 15)
```

### Content Blocked
```
❌ 错误: Content blocked
👉 [在浏览器中手动处理]
👉 输入: Enter (继续)
```

### 手动干预
```
❌ 错误: 找不到按钮
👉 [在浏览器中手动操作]
👉 输入: Enter (继续)
```

### 跳过问题视频
```
❌ 错误: 视频格式不支持
👉 输入: skip
```

## 🔁 批量处理流程

```
1. 准备第一批视频（更新 VideoList.csv）
   ↓
2. 运行程序: python video_automation.py
   ↓
3. 处理完成
   ↓
4. 准备下一批视频（更新 VideoList.csv）
   ↓
5. 输入: continue
   ↓
6. 重复步骤 3-5
   ↓
7. 全部完成，输入: quit
```

---

## 💡 处理策略

### 小批次（3-5个）
- ✅ 网络不稳定
- ✅ 首次测试
- ✅ 复杂视频

### 中批次（10个）
- ✅ 网络稳定
- ✅ 常规处理
- ✅ 平衡效率

### 大批次（20个）
- ✅ 网络很稳定
- ✅ 简单视频
- ✅ 最大效率

---

## 🔧 错误处理流程

```
遇到错误
  ↓
1. 尝试 retry
  ↓ 失败
2. 检查浏览器
  ↓ 发现问题
3. 手动处理 + Enter
  ↓ 仍失败
4. 输入步骤号重试
  ↓ 还是失败
5. 输入 skip 跳过
```
